# Lo-fi Prototypes: Flow "Ключи" — Design Document

**Дата:** 2026-02-21
**Проект:** Домофон Спутник с дисплеем (Tor display)
**Экран:** 2.83" TFT LCD, 480x640px (портрет), цветной, не сенсорный

---

## Контекст

Новая модель домофона «Спутник» — первая с экраном. Подрядчики (дизайн-студия) разрабатывают UI и просят Lo-fi прототипы экранов по утверждённым блок-схемам. Формат — в стиле предоставленных примеров: экраны с обязательными/опциональными элементами, фоновые процессы, карточки вопросов и дополнительной информации.

## Формат доставки

HTML-wireframes → импорт в Figma через плагин "html.to.design"

- Одна HTML-страница со всеми экранами и flow-диаграммой
- Экраны в масштабе 480x640px
- Lo-fi стиль: wireframe, без цветового оформления, текстовые placeholder-ы
- Стрелки/связи между экранами показывают переходы
- Фоновые процессы выделены голубым фоном
- Карточки вопросов и доп. информации рядом с экранами

---

## Структура Flow

```
Экран ожидания
  ├── [периодически] → Экран ожидания + рекламный блок → возврат
  │
  └── [ключ приложен] → [ФОН: Проверка регистрации в МП]
      ├── НЕТ регистрации → Отказано в доступе → Экран ожидания
      │
      └── ЕСТЬ → [ФОН: Проверка валидности ключа]
          ├── НЕВАЛИДНЫЙ → Отказано в доступе → Экран ожидания
          │
          └── ВАЛИДНЫЙ → [ФОН: Тип ключа?]
              ├── СЕРВИСНЫЙ → Открыто сервисным ключом → Экран ожидания
              │
              └── ОБЫЧНЫЙ/ДЕТСКИЙ → [ФОН: Показать промо?]
                  ├── ДА → Промо-экран функции МП → далее по flow
                  └── НЕТ → [ФОН: Показать информирование?]
                      ├── ДА → Задолженность / Ключ скомпрометирован
                      │         → Открыто ключом/детским → Экран ожидания
                      └── НЕТ → Открыто ключом/детским → Экран ожидания
```

## Экраны

### 1. Стандартный экран ожидания

**Обязательные элементы:**
- Адрес (улица, дом), номер подъезда
- Диапазон квартир (напр. «кв. 1–120»)
- Статус системы («Онлайн» / «Оффлайн»)
- Предложение действия: «Приложите ключ или введите номер квартиры»

**Опциональные элементы:**
- Часы и дата
- Логотип и/или контакты управляющей компании
- Информация о погоде (температура, иконка)

**Звуковая индикация:** нет

---

### 2. Экран ожидания + рекламный блок

**Обязательные элементы:**
- Рекламный контент (видео 3–5 сек / статичное изображение / бегущая строка)
- Примеры: услуги ЖКХ, доставка, локальные сервисы, QR-коды

**Опциональные элементы:**
- Адрес, номер подъезда (может скрываться)
- Звуковая индикация / озвучивание рекламы

**Дополнительная информация:**
- Запрет рекламы в ночной период
- Запрет рекламы при ЧС
- Запрет рекламы при отсутствии интернета
- Звуковое сопровождение ночью не проигрывается

**Переходы:** после проигрывания → Экран ожидания

**Вопросы:**
- Какой интервал между рекламными блоками?
- Какой максимальный хронометраж рекламного блока?
- Может ли пользователь прервать рекламу прикладыванием ключа?

---

### 3. Открыто ключом

**Обязательные элементы:**
- Текстовый заголовок: «Входите, дверь открыта»
- Статус успеха (иконка — зелёная галочка / открытый замок)
- Таймер обратного отсчёта («Дверь закроется через 5 сек»)

**Опциональные элементы:**
- Номер квартиры
- Имя/приветствие владельца ключа

**Звуковая индикация:**
- Звук успешного доступа
- Опционально: озвучивание заголовка

**Переходы:** через X мс → Экран ожидания

---

### 4. Открыто детским ключом

**Обязательные элементы:**
- Текстовый заголовок: «Входите, дверь открыта»
- Статус успеха (иконка)
- Таймер обратного отсчёта

**Опциональные элементы:**
- Номер квартиры

**Звуковая индикация:**
- Звук успешного доступа

**Дополнительная информация:**
- Не показываем рекламу при открытии детским ключом
- Не показываем информирование о задолженности

**Вопросы:**
- Визуально отличается от обычного «Открыто ключом»?
- Нужна ли push-уведомление родителю?

**Переходы:** через X мс → Экран ожидания

---

### 5. Открыто сервисным ключом

**Обязательные элементы:**
- Текстовый заголовок: «Открыто сервисным ключом»
- Статус (иконка — ключ / сервисный символ)
- Таймер

**Опциональные элементы:**
- Идентификатор сервисной организации

**Звуковая индикация:**
- Звук доступа

**Дополнительная информация:**
- Не показываем рекламу
- Не показываем информирование
- Логируется отдельно

**Переходы:** через X мс → Экран ожидания

---

### 6. Отказано в доступе

**Обязательные элементы:**
- Текстовый заголовок: «Доступ запрещён»
- Статус блокировки (иконка — красный крест / закрытый замок)
- Таймер (через X мс → Экран ожидания)

**Опциональные элементы:**
- Текст пояснения отказа («Ключ не зарегистрирован»)
- Контакты для уточнения (УК / техподдержка)

**Звуковая индикация:**
- Звук отказа в доступе
- Опционально: озвучивание заголовка

**Вопросы:**
- Разные тексты для «невалидный ключ» и «нет регистрации»?

**Переходы:** через X мс → Экран ожидания

---

### 7. Информирование о задолженности за домофон

**Обязательные элементы:**
- Текстовый заголовок: «Задолженность за домофон»
- Текст пояснения (сумма, период)
- Таймер (через X сек → «Открыто ключом»)

**Опциональные элементы:**
- Контакты для уточнения (телефон УК)
- QR-код для перехода к оплате
- Номер квартиры

**Звуковая индикация:**
- Звук информационного сообщения

**Дополнительная информация:**
- НЕ блокирующий — после таймера дверь откроется
- Не показываем при детском / курьерском / сервисном ключе

**Вопросы:**
- Какие типы/виды задолженностей?
- Показываем конкретную сумму или только факт?
- Какой источник данных о задолженности?

---

### 8. Информирование о задолженности за ЖКХ

Аналогичен экрану 7.

**Заголовок:** «Задолженность за ЖКХ»

**Вопросы:**
- Могут ли обе задолженности показываться одновременно?
- Последовательно? Приоритет?

---

### 9. Ключ скомпрометирован

**Обязательные элементы:**
- Текстовый заголовок: «Ключ скомпрометирован»
- Текст пояснения: «Обратитесь в управляющую компанию»
- Статус предупреждения (иконка)

**Опциональные элементы:**
- Контакты УК / техподдержки
- QR-код на приложение

**Звуковая индикация:**
- Звук предупреждения

**Вопросы:**
- Открывается ли дверь скомпрометированным ключом? Или блокирующий?

---

### 10. Промо-экран функции МП

Шаблон для рекламы функций мобильного приложения. Типы промо по блок-схеме:
- Архив глубиной 3 дня
- Бесключевой доступ
- Многоадресность
- Интеграция с Алисой
- Расширенная история посещений
- Код доступа
- Мгновенная загрузка
- Видео в онлайн
- Приём звонков
- История посещений
- Открытие двери в приложении

**Обязательные элементы:**
- Заголовок функции (напр. «Откройте дверь прямо из приложения»)
- Иллюстрация / иконка функции
- QR-код для скачивания приложения

**Опциональные элементы:**
- Краткое описание выгоды
- Логотип приложения «Спутник»

**Звуковая индикация:** нет

**Дополнительная информация:**
- Показываем только если нет регистрации в МП
- Показываем после валидации, перед экраном «Открыто»
- Один промо за одно открытие (ротация)
- Таймер: через X сек → следующий экран

## Фоновые процессы

| # | Процесс | Условие | Да | Нет |
|---|---------|---------|-----|------|
| 1 | Проверка регистрации в МП | Есть регистрация? | → Проверка валидности | → Отказано в доступе |
| 2 | Проверка валидности ключа | Ключ валидный? | → Проверка типа ключа | → Отказано в доступе |
| 3 | Проверка типа ключа | Сервисный? | → Открыто сервисным | → Показать промо? |
| 4 | Показать промо? | Да + нет рег. в МП | → Промо-экран | → Показать информирование? |
| 5 | Показать информирование? | Есть долг/компрометация | → Информирование | → Открыто ключом/детским |
